# Page Override: Итоги проверки

> Переопределяет MASTER.md для экрана завершения обхода.

## Специфика экрана

Экран «победы» — техник завершил обход. Должен давать чёткое ощущение завершённости
и направлять к следующему шагу (синхронизации). Минимум элементов, максимум ясности.

## Layout

```
┌─────────────────────────────────────────────────┐
│ ← Итоги обхода                                 │ ← App Bar
│                                                 │
│                   ✓                             │ ← иконка 64dp, success
│            Обход завершён                       │ ← displayLarge, textPrimary
│          04 апр 2026 · 14:32                    │ ← bodyMedium, textSecondary
│                                                 │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────┐ │
│  │      12      │ │      3       │ │    8     │ │ ← stat cards
│  │   Проверено  │ │   Дефектов   │ │   Фото   │ │
│  └──────────────┘ └──────────────┘ └──────────┘ │
│                                                 │
│  ────────────────────────────────────────────   │
│                                                 │
│       [  Синхронизировать  ]   ← Primary        │
│       [  Сохранить отчёт   ]   ← Secondary      │
│       [  К списку маршрутов]   ← Ghost           │
│                                                 │
└─────────────────────────────────────────────────┘
```

## Иконка завершения

- Размер: 64dp.
- Иконка: `check_circle` из Material Symbols.
- Цвет: `color.success`.
- Анимация появления: scale 0.5→1.0 + fade-in, 300ms ease-out.
  Уважает `isReduceMotionEnabled` (без анимации).

## Stat Cards (статистика)

- 3 карточки в ряд, равная ширина (flex:1), gap 8dp.
- Padding: 12dp, radius: `radius.md`.
- Background: `color.surface`.
- Число: `type.displayMedium` (20sp, 600), `color.textPrimary`.
- Лейбл: `type.caption` (12sp), `color.textSecondary`.
- Карточка «Дефектов» при count > 0: число `color.error`.
- Карточка «Фото» при count > 0: иконка камеры 14dp рядом с числом.

## Состояние «Offline / Ожидает синхр.»

```
┌─ Warning banner ────────────────────────────────┐
│ ⇅  Данные сохранены локально · Ожидают отправки │
└─────────────────────────────────────────────────┘
```

- Показывается над кнопками если нет сети.
- Background: `color.syncSubtle`, text: `color.sync`, иконка `sync` 16dp.
- Кнопка «Синхронизировать»: остаётся enabled (запустит при появлении сети).

## Ошибка синхронизации

```
┌─ Error banner ──────────────────────────────────┐
│ ✕  Ошибка синхронизации · Попробуйте ещё раз   │
│                              [Повторить]        │
└─────────────────────────────────────────────────┘
```

- Показывается вместо success-баннера.
- Background: `color.errorSubtle`, border: 1dp `color.error`.
- Кнопка «Повторить» внутри баннера: Ghost, `color.error`.

## Навигация

- «Назад» в App Bar: ведёт к списку маршрутов (не обратно в маршрут).
- «К списку маршрутов» Ghost-кнопка: то же действие, visually de-emphasized.
- После успешной синхронизации: SnackBar «Данные отправлены ✓» (auto-dismiss 3s),
  маршрут в списке обновляет статус на «Завершён».
