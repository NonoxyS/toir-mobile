# Page Override: Экран маршрута (точки обхода)

> Переопределяет MASTER.md для экрана списка точек маршрута.

## Специфика экрана

Навигационный экран между списком маршрутов и чек-листом. Техник видит все точки,
понимает прогресс и переходит к конкретной точке. Ключевое: «где я сейчас?».

## Структура экрана

```
┌─ App Bar ──────────────────────────────────────┐
│ ← ПТО-7 · Насосная ст. №3      [⋮ меню]      │
└────────────────────────────────────────────────┘
  ─ Progress header ─────────────────────────────
  │ Точка 3 из 12   ████████░░░░░░  25%         │
  ───────────────────────────────────────────────

  ─── Текущая точка ───────────────────────────── (заголовок секции)
  ┌─ Highlighted card ────────────────────────┐
  │ ▶ Компрессор К-1 · Зона А               │  ← headline, textPrimary
  │   ID: PT-047                              │  ← caption, textSecondary
  │   [Открыть чек-лист]   [Скан QR]        │
  └───────────────────────────────────────────┘

  ─── Следующие точки ─────────────────────────
  ┌─────────────────────────────────────────────┐
  │ ○ Насос Н-2 · Зона Б          [Не начата]  │
  │ ○ Вентилятор В-3 · Зона В     [Не начата]  │
  └─────────────────────────────────────────────┘

  ─── Завершённые ────────────────────────────── (свёрнуто по умолчанию)
  │ ✓ Задвижка З-1               [Выполнена]   │
  │ ✓ Клапан К-6                 [Выполнена]   │

┌─ Sticky footer ────────────────────────────────┐
│   [Завершить обход]  ← появляется после послед.│
└────────────────────────────────────────────────┘
```

## Progress Header

- Pinned под App Bar (не sticky при скролле — всегда виден).
- Height: 56dp, background `color.background`, border-bottom 1dp `color.borderSubtle`.
- Текст «Точка X из Y»: `type.bodyMedium`, `color.textSecondary`.
- Прогресс-бар: full-width минус 2×spacing.md, height 4dp.
- Процент: `type.bodyMedium` (500), `color.textPrimary`, справа.

## Карточка текущей точки (Highlighted)

- Background: `color.surface2` + border 1.5dp `color.border` (светлее стандарта для выделения).
- Левый accent bar: 4dp `color.warning` (точка «в процессе»).
- Padding: 16dp.
- Кнопки внутри: row, gap 8dp.
  - «Открыть чек-лист»: Primary, flex:1.
  - «Скан QR»: Secondary, width 48dp (только иконка QR).

## Строки обычных точек

- Height: 56dp, padding 16dp горизонтальный.
- Разделитель: 1dp `color.borderSubtle`.
- Layout: [иконка статуса 20dp] [название] [бейдж статуса] [стрелка →].
- Завершённые точки: свёрнуты в коллапсируемую секцию «Завершённые (N)».
  - Тап на заголовок секции → разворачивает.
  - Внутри: opacity 0.7, иконка ✓ зелёная.

## Иконки статуса точки

| Статус | Иконка | Цвет |
|--------|--------|------|
| Не начата | `radio_button_unchecked` | textDisabled |
| В процессе | `play_circle` | warning |
| Завершена | `check_circle` | success |
| Пропущена | `cancel` | error |

## QR-сканирование

- Иконка QR в карточке текущей точки.
- Тап → открывает нативный сканер (bottom sheet или full-screen камера).
- При успешном скане → подсветка «Точка подтверждена ✓», переход в чек-лист.
- При ошибке → SnackBar «Неверный QR-код».

## Кнопка «Завершить обход»

- Появляется в sticky footer только после того как последняя точка выполнена.
- Primary, full-width.
- Тап → экран «Итоги проверки».
