# Page Override: Авторизация

> Переопределяет MASTER.md для экрана входа в систему.

## Специфика экрана

Единственный экран с логотипом. Простой, без навигации, фокус на надёжности.
Нет bottom navigation. Нет Top App Bar с «назад».

## Layout

## Детали

### Логотип / заголовок

- Название «TOIR» или логотип: `type.displayLarge` (24sp, 600), `color.textPrimary`
- Подзаголовок: `type.bodyMedium`, `color.textSecondary`
- Вертикальное центрирование в верхней трети экрана

### Поля ввода

- Стандартные TextField из MASTER.md
- Пароль: показатель toggle (иконка 👁), `color.textSecondary`
- Auto-focus на поле «Логин» при открытии экрана
- Return key на «Логин» → фокус на «Пароль»
- Return key на «Пароль» → trigger входа

### Состояния ошибок

- Неверный логин/пароль: оба поля рамка `color.error` + текст под паролем:
  «Неверный логин или пароль» — `type.caption`, `color.error`
- Нет сети: текст «Нет подключения к сети» — `type.caption`, `color.warning`
  + кнопка «Войти» disabled
- Сервер недоступен: «Сервер недоступен. Попробуйте позже» — `color.error`

### Кнопка «Войти»

- Width: fixed, max 280dp, centered
- Loading state: кружок вместо текста, disabled
- Не full-width (экран авторизации не производственный, более «сдержанный» стиль)

### «Забыли пароль?»

- `type.caption` (12sp), `color.textSecondary`, underline, center
- Открывает системный браузер или контакт диспетчера (согласно backend API)

## Клавиатура

- При появлении клавиатуры: экран сжимается, кнопка «Войти» остаётся видимой
- `KeyboardAvoidingView` с behavior `padding`
